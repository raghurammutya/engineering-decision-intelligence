# ML Pilot

The ML repository is the first pilot/customer system for Engineering Decision
Intelligence.

## Pilot Role

The ML repository should provide:

- real workflow and script surfaces,
- current environment promotion policy,
- existing evidence and registry examples,
- examples of operational state mutation paths,
- examples of stale, duplicate, or non-canonical automation.

## Product Boundary

Reusable product logic belongs in this repository.

ML-specific generated outputs, evidence packs, and operational findings belong
in the ML repository unless they are being intentionally promoted into reusable
examples.

## Initial Pilot Question

Which ML workflows and scripts can change operational state, and are they
canonical, owner-approved, evidenced, and safe?
